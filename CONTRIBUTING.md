# Contributing Guidelines

Thank you for your interest in contributing to the DevOps Project! This document provides guidelines for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes and commit with clear messages
5. Push to your fork and submit a pull request

## Code Standards

- Follow existing code style and formatting
- Write meaningful commit messages
- Ensure all tests pass before submitting PRs
- Add tests for new functionality

## Development Setup

Refer to `STARTUP_GUIDE.md` for detailed setup instructions for your development environment.

## Testing

- Frontend: `npm test` in the `frontend/` directory
- Shopping Cart: `npm test` in the `shopping-cart/` directory
- Product Catalog: `pytest` in the `product-catalog/` directory

## Pull Request Process

1. Update documentation if needed
2. Add tests for your changes
3. Ensure CI/CD pipeline passes
4. Request review from maintainers

## Questions or Issues?

Feel free to open an issue on GitHub for bugs, feature requests, or general questions.
